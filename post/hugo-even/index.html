Use Hugo+Even+Github to build a personal blog

2021-04-08
Tutorial
386 words 2 mins read

  • 3.add blog theme
  • 4.start hugo

1.install Hugo

sudo apt install hugo

2.build hugo project

hugo new site [project-name] e.g. my site is “blog” which is created by command “hugo new site blog”

3.add blog theme

enter site directory: cd blog add theme: git clone https://github.com/olOwOlo/hugo-theme-even themes/even

4.start hugo

hugo server

5.configure theme

For personal use, we need to modify the config.toml file, just change the configuration according to the instructions. My theme configuration file https://github.com/zhzhwei/zhzhwei.github.io/blob/master/config.toml Go to the blog/themes/even folder and you will see that the file structure is almost identical to the one of the new Hugo project. This is set up so that the user’s configuration can override the theme’s configuration. For example, if I want to customize the bottom display, the hugo-theme-even bottom configuration is controlled by blog/themes/even/layouts/partials/footer.html. To override the theme’s configuration, create a new blog/layouts/partials/footer.html file in the root of the project and fill in the custom content to override the theme’s configuration. The overriding is the same for other files. hugo-theme-even uses Webpack to package js,css and add hash value to the file name, this kind of css,js files can’t be overwritten, but hugo-theme-even configuration file has parameters to overwrite this css,js, refer to my file configuration for details.

6.add a new blog

The command to add a new blog is relatively simple, it’s as follows: “hugo new post/my-first-blog.md” copy this command then it will use the template to create a file. It looks for the user’s template file first, if template file does’t exist, it will look for the theme’s template file. The hugo-theme-even template file “blog/themes/even/archetypes/default.md” is relative complicated, just use the “blog/archetypes/default.md” file to overwrite it. My template configuration is as follows:

The draft parameter value controls whether the page is displayed on the website. The page will be displayed only if it is set to false or if the page is deleted. The previous content before <!--more--> is automatically used as a basic information of page.

7. package md file

In order to deploy online, you need to package the Markdown file into an HTML file. The packaging command is as follows: “hugo -t even”, “even” is the subject name.
