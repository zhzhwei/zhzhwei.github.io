[Tutorial] Use Hugo+Even+Github to build a personal blog

1.install Hugo

command: sudo apt install hugo

2.build hugo site

command: “hugo new site [site-name]”

eg: my site is “blog” which is created by command “hugo new site blog”.

After installation you can verify it by entering the following command: “hugo version”.

Once created, the following file structure will be created under the blog folder.

3.add blog theme

enter site directory: “cd blog”

and then add theme: “git clone https://github.com/olOwOlo/hugo-theme-even themes/even”.

Enter the “blog/themes/even/exampleSite” folder,

copy the config.toml file and content folder to overwrite the config.toml and content folder int the root of the site.

4.start hugo

command: hugo server

Enter the link http://localhost:1313/ in your browser to see the results.

5.configure theme

For personal use, we need to modify the config.toml configuration file according to the instructions. My theme configuration file “https://github.com/zhzhwei/zhzhwei.github.io/blob/master/config.toml". Enter “blog/themes/even” folder and you will see that the file structure is almost identical to the one of the new Hugo site. This setting is so that the user’s configuration can override the theme’s configuration. For example, if I want to customize the bottom display, the hugo-theme-even bottom configuration is controlled by “blog/themes/even/layouts/partials/footer.html”. To override the theme’s configuration, create a new “blog/layouts/partials/footer.html” file in the root of the site and fill in the custom content to override the theme’s configuration. The overriding is the same for other files. hugo-theme-even uses Webpack to package js,css and add hash value to the file name, these css,js files can’t be overwritten, but hugo-theme-even configuration file has parameters to overwrite this css,js, refer to my theme configuration file for more details.

6.add a new blog

command: hugo new post/hugo-even.md

It will automatically look for the post folder unter site and then use the template to create a file. It will try to use user’s template file first, if this file does’t exist, it will look for the theme’s template file. The hugo-theme-even template file “blog/themes/even/archetypes/default.md” is relatively complicated, just use the “blog/archetypes/default.md” file to overwrite and edit it. My template default.md is as follows:

---
title: "{{ replace .TranslationBaseName "-" " " | title }}"
date: {{ .Date }}
draft: false
tags: []
categories: []
---
<!--more-->

The draft parameter value controls whether the page is displayed on the website. The page will be displayed only if it is set to false or if the page is deleted. The previous content before <!--more--> is automatically used as a basic information of page.

7.package md file

In order to deploy online, you need to package the Markdown file into an HTML file. Enter the command: “hugo -t even” in the root directory, “even” is the subject name, then the “public” folder will be created in the blog folder. As like Hexo, the contents of Hugo’s public folder are the files for the static site.

8.create a new repo

Create a new repository with username.github.io (eg.zhzhwei.github.io). Clone this repo from github website to your computer. Delete all the files except “.git” in your computer.

9.deploy online

Copy all the files except “.git” folder from hugo site (eg.blog) to the repo on it(eg:zhzhwei.github.io) and then begin to deploy online. Finally, open username.github.io in your browser and you will see your generated personal blog. My commands are as follows:
